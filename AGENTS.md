# AGENTS.md — AI Coding Agent Instructions

This file contains instructions for AI coding agents (Cursor, Copilot, Claude, etc.)
working on this repository.

## Repository Overview

Python 3 template using: **UV** (packaging) · **Ruff** (lint + format) · **Ty** (type checking) · **pytest** (testing) · **Mise** (tasks + tools)

## Project Structure

```
.
├── .mise/tasks/          # Mise task files (all tasks live here as scripts)
├── .github/
│   ├── workflows/        # GitHub Actions CI/CD pipelines
│   ├── dependabot.yml    # Dependabot config
│   └── CODEOWNERS        # Code ownership
├── config/
│   └── githooks/hooks/   # Git hooks (managed via mise run hooks:sync)
├── scripts/              # Shared bash utilities
├── src/
│   └── python_template/  # Main Python package (rename for your project)
│       ├── __init__.py
│       ├── __main__.py   # CLI entry point
│       ├── version.py    # Auto-generated — do NOT edit manually
│       └── py.typed      # PEP 561 marker
├── tests/                # pytest test suite
├── mise.toml             # Tool versions + task discovery
├── pyproject.toml        # All Python config (Ruff, pytest, coverage, etc.)
└── CHANGELOG.md          # Changelog (keep updated on user-facing changes)
```

## Important Rules

1. **Never edit `src/python_template/version.py` manually** — it is generated by `mise run build`
2. **Never merge branches** — always rebase: `mise run vcs:rebase`
3. **Version bumps** use `mise run bump:patch/minor/major` which commits + creates git tags
4. **Push tags separately** with `mise run tag:push` after all commits are ready

## Running Tasks

```bash
mise run install        # install all dependencies
mise run ci:all         # full CI pipeline
mise run lint           # ruff check
mise run lint:fix       # ruff check --fix
mise run fmt            # ruff format
mise run fmt:check      # ruff format --check
mise run typecheck      # ty check
mise run test           # pytest
mise run test:cov       # pytest + coverage
mise run bump:patch     # bump patch version
mise run bump:minor     # bump minor version
mise run bump:major     # bump major version
mise run tag:push       # push tags → triggers release
mise run hooks:sync     # sync git hooks from orchestras/git-hooks
mise run scan:deps      # audit dependencies
mise run scan:secrets   # scan for leaked secrets
```

## Adding New Tasks

Create a new file in `.mise/tasks/<name>` with a bash shebang and the
`# mise description="..."` comment on line 2. Make it executable:
```bash
chmod +x .mise/tasks/<name>
```

For namespaced tasks (e.g. `foo:bar`), create `.mise/tasks/foo:bar`.

## Code Quality Standards

- All public functions must have **Google-style docstrings**
- All functions must have **type annotations**
- Minimum Python version: **3.12**
- Line length: **100**
- Run `mise run ci:all` before committing

## Security

- Use `mise run scan:deps` to audit dependencies before releases
- Use `mise run scan:secrets` to check for leaked secrets
- GHAS (CodeQL) runs weekly and on every push to main/develop
- Bandit runs on every PR as part of the security check

## Release Process

1. Make changes on a feature branch
2. `git rebase origin/develop` (or `mise run vcs:rebase`)
3. Open PR targeting `develop` — required checks must pass
4. Merge to `develop` (linear/rebase only)
5. When ready to release: `mise run vcs:release` (develop → main)
6. On main: `mise run bump:patch` (or minor/major) → `mise run tag:push`
7. CI automatically builds PyInstaller binaries for all platforms and creates a GitHub Release
